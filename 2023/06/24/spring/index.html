POJO是指”Plain Old Java Object”，即”普通的旧Java对象”。它是指一个普通的Java对象，不继承或实现任何框架或类库提供的特殊接口或类，也不依赖于任何特殊的技术，如EJB（Enterprise Java Bean）等。POJO通常只包含一些基本的属性和getter/setter方法，用于封装数据和提供访问数据的接口。

POJO的概念最初是由Martin Fowler提出的，旨在强调使用简单的Java对象来代替复杂的、依赖于特定框架的Java对象，从而提高代码的可读性、可维护性和可测试性。

在Java开发中，POJO通常用于表示业务数据、数据传输对象（DTO）、值对象（VO）等。它们的设计目的是为了简化代码，降低耦合性，提高代码的可重用性和可测试性。

Core Container:

  • Core Container (核心容器)包含Core、Beans、Context和Express Language模块。
  • Core 和Beans模块是框架的基础部分，提供IoC(控制反转)和依赖注入。
  • 这里的基础概念是BeanFactory，它提供对Factory模式的经典实现来消除对程序性单例模式的需要，并真正允许你从程序逻辑中分离出依赖关系和配置。
  • Core模块主要包含Spring框架中基本的核心工具类，Spring的其他组件都要使用这个包里面的类，Core模块是其他组件的基本核心。当然你也可以在自己的应用系统中使用这些工具类。
  • Beans模块是所有应用都要用到的，它包含访问配置文件、创建和管理bean以及进行Inversion of Control/Development Injection（IoC/DI）操作相关的所有类。
  • Context模块构建于Core和Beans模块基础之上，提供了一种类似于JNDI注册器的框架式的对象访问方法。Context模块集成了Beans的特性，为Spring核心提供了大量扩展，添加了对国际化（例如资源绑定）、事件传播、资源加载和对Context的透明创建的支持。Context模块同时也支持J2EE的一些特性，例如EJB、JMX和基础的远程处理。ApplicationContext接口是Context模块的关键。
  • Expression Language模块提供了一个强大的表达式语言用于在运行时查询和操纵对象，它是JSP2.1规范中定义的unified expression language的一个扩展，该语言支持设置/获取属性的值，属性的分配，方法的调用，访问数组上下文（accessiong the context of arrays）、容器和索引器、逻辑和算术运算符、命名变量以及从Spring的IoC容器中根据名称检索对象。它也支持list投影、选择和一般的list聚合。
